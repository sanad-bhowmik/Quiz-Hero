  • Different between Map,Filter, For Each and Find

    Map, Filter, For Each, and Find are methods commonly used with arrays in web development.

    1. 1.Map creates a new array by applying a function to each element of an existing array.
    2. 2.Filter creates a new array by selecting elements from an existing array that meet a certain condition.
    3. 3.For Each executes a provided function for each element of an array.
    4. 4.Find returns the value of the first element in an array that satisfies a given condition.

  • Different between Local storage, section storage and cookies.

    Local storage, session storage, and cookies are all ways to store data in the browser in web development.

    1. 1. Local storage is a way to store data on the client-side browser indefinitely, even after the browser is closed. The data is accessible across all pages of a domain and has no expiration date unless manually cleared.
    2. 2. Session storage is similar to local storage but the data is only accessible during the current browser session, meaning it will be deleted when the user closes the browser.
    3. 3. Cookies are small text files that are stored on the user's computer and can be used to store data for a specific website. Cookies have an expiration date and can be accessed by the server and the client-side JavaScript.

  • What is event loop and how it works?

    The event loop is a fundamental mechanism of JavaScript that handles the execution of code in a non-blocking and asynchronous way. It continuously listens for events, such as user input or network responses, and pushes event callbacks into a queue.
    Here's how it works:

    1. 1. JavaScript code is executed in a single thread by the JavaScript runtime.
    2. 2. When an asynchronous event, such as a mouse click or network request, occurs, the corresponding event handler is added to a queue by the browser or the runtime.
    3. 3. The event loop continuously checks this queue for new events or callbacks to be executed.
    4. 4. If the call stack is empty, the event loop takes the first event or callback from the queue and pushes it onto the call stack for execution.
    5. 5. Once the callback or event handler is executed, it is removed from the call stack and the event loop moves on to the next item in the queue.
    6. 6. If the call stack is not empty, the event loop continues to check the queue for new events, but does not add any new items to the call stack until it is empty.

  • Quiztime function works

    The quizTimer function takes a dismiss parameter, which is used to stop the timer if it is set to true. Otherwise, it creates an interval timer using setInterval that updates an HTML element with an ID of "count" every second. Inside the interval timer, it calculates the minutes and seconds based on the current count value, and formats them with leading zeros if necessary. It then updates the "count" HTML element with the formatted time and increments the count value. If the count value is greater than 60, the function alternates the text color of the "count" HTML element every two seconds between red-300 and red-700 to provide a visual cue to the user.
